SIP-Ruang & Sarpras

Platform Desktop berbasis Client-Server (Offline) untuk digitalisasi peminjaman fasilitas kelas dan sarana prasarana kampus.

Glosarium & Istilah Teknis Project

  • TU: Tata Usaha (Admin Pengelola Server)
  • Mhs: Mahasiswa (Client / Pengguna Kiosk)
  • Kiosk: Terminal/Aplikasi mandiri untuk mahasiswa
  • CRUD: Create, Read, Update, Delete
  • ACC: Accepted (Pengajuan disetujui Admin)
  • Pending: Status menunggu keputusan TU
  • Qty: Quantity (Jumlah unit barang)
  • PK (Primary Key): Kunci identitas unik di tabel
  • FK (Foreign Key): Kunci relasi antar tabel
  • JasperReports: Library Java untuk men-generate PDF

-- Pembuatan Entitas Dasar Database
CREATE DATABASE db_sarpras_kampus;
USE db_sarpras_kampus;

-- Tabel [1/5]: Administrator (Server) - PK: id_admin
CREATE TABLE tb_admin_tu (
    id_admin INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    password VARCHAR(50) NOT NULL,
    nama_staff VARCHAR(100)
);

-- Tabel [2/5]: Mahasiswa (Client) - PK: npm
CREATE TABLE tb_mahasiswa (
    npm VARCHAR(20) PRIMARY KEY,
    nama_mhs VARCHAR(100) NOT NULL,
    password VARCHAR(50) NOT NULL,
    prodi VARCHAR(50),
    no_telp VARCHAR(15)
);

-- Tabel [3/5]: Fasilitas Ruangan - PK: id_ruang
CREATE TABLE tb_ruangan (
    id_ruang VARCHAR(10) PRIMARY KEY,
    nama_ruang VARCHAR(100) NOT NULL,
    kapasitas INT,
    fasilitas VARCHAR(100)
);

-- Tabel [4/5]: Fasilitas Inventaris - PK: id_barang
CREATE TABLE tb_barang (
    id_barang VARCHAR(10) PRIMARY KEY,
    nama_barang VARCHAR(100) NOT NULL,
    stok_total INT
);

-- Tabel [5/5]: Transaksi Multi-Item - PK: id_trx | FK: npm
CREATE TABLE tb_transaksi (
    id_trx INT AUTO_INCREMENT PRIMARY KEY,
    npm VARCHAR(20),
    tipe_pinjam ENUM('RUANG', 'BARANG'),
    id_item VARCHAR(10),
    qty INT DEFAULT 1,
    tgl_pinjam DATE,
    jam_mulai TIME,
    jam_selesai TIME,
    keterangan TEXT,
    status_acc VARCHAR(20) DEFAULT 'Pending',
    FOREIGN KEY (npm) REFERENCES tb_mahasiswa(npm) ON DELETE CASCADE
);

-- ========================================================
-- Bonus Level 3: TRIGGER MYSQL OTOMATISASI STOK BARANG
-- ========================================================
DELIMITER $$

CREATE TRIGGER trg_update_stok_barang
AFTER UPDATE ON tb_transaksi
FOR EACH ROW
BEGIN
    -- Pastikan trigger HANYA berjalan jika tipe pinjamnya adalah 'BARANG'
    IF NEW.tipe_pinjam = 'BARANG' THEN
        
        -- KONDISI 1: Admin mengubah status dari 'Pending' menjadi 'acc' (Barang keluar / dipinjam)
        IF OLD.status_acc = 'Pending' AND NEW.status_acc = 'acc' THEN
            UPDATE tb_barang 
            SET stok_total = stok_total - NEW.qty 
            WHERE id_barang = NEW.id_item;
        END IF;

        -- KONDISI 2: Admin membatalkan status dari 'acc' kembali ke 'Pending' (Barang batal dipinjam / dikembalikan)
        IF OLD.status_acc = 'acc' AND NEW.status_acc = 'Pending' THEN
            UPDATE tb_barang 
            SET stok_total = stok_total + NEW.qty 
            WHERE id_barang = NEW.id_item;
        END IF;

    END IF;
END$$

DELIMITER ;

package koneksi;
import java.sql.Connection;
import java.sql.DriverManager;

public class KoneksiDB {
    private static Connection koneksi;
    
    public static Connection getKoneksi() {
        if (koneksi == null) {
            try {
                String url = "jdbc:mysql://localhost:3306/db_sarpras_kampus";
                String user = "root"; 
                String pass = "";     
                
                DriverManager.registerDriver(new com.mysql.cj.jdbc.Driver());
                koneksi = DriverManager.getConnection(url, user, pass);
                System.out.println("Sukses: Database Terhubung!");
            } catch (Exception e) {
                System.out.println("Gagal: " + e.getMessage());
            }
        }
        return koneksi;
    }
}

Saran Pengembangan Project (Bonus)

Tips khusus aplikasi Offline & Visual untuk memukau Dosen Penguji.

Karena ini adalah project mata kuliah Pemrograman Visual dengan sifat Offline, fokus utama untuk mendapatkan nilai "A" atau pujian dari dosen adalah dengan memaksimalkan Eksplorasi Komponen GUI (Swing), Interaktivitas, dan Manipulasi Visual. Namun, memperkuat sisi Database akan menyempurnakan project ini dari segi performa dan keamanan data. Berikut adalah fitur "Wah" yang 100% offline dan sangat relevan untuk project ini:

🌟 Level 1: "Eye-Candy" & Manipulasi UI Dasar (Wajib Dicoba)

Fokus pada hal-hal yang langsung terlihat saat aplikasi baru dijalankan.

Gunakan Tema FlatLaf (Flat Look and Feel)

Mengapa: Desain bawaan Java (Nimbus/Metal) terlihat sangat jadul. FlatLaf membuat aplikasi Anda terlihat seperti aplikasi Windows 11 modern.
Nilai Plus Visual: Anda bisa menambahkan tombol "Mode Gelap" (Dark Mode) di pojok aplikasi yang mengubah seluruh warna form hanya dengan 1 baris kode.

Splash Screen dengan JProgressBar

Sebelum masuk ke Form Login, buat satu Form Splash Screen kecil tanpa border (Undecorated). Beri gambar logo kampus/project, lalu jalankan JProgressBar (loading dari 0% ke 100%) menggunakan Timer. Setelah 100%, baru form Login muncul. Ini sangat klasik tapi selalu berhasil memberi kesan profesional.

Kustomisasi Tabel (JTable dengan Tombol)

Alih-alih membuat form terpisah atau klik baris lalu klik tombol di luar, buat Tombol Aksi (ACC/Tolak) berada di DALAM sel tabel. Ini membutuhkan manipulasi TableCellRenderer dan TableCellEditor di Java Swing, yang menunjukkan pemahaman tingkat lanjut Anda tentang komponen Visual.

🚀 Level 2: Eksplorasi Komponen Visual Lanjutan

Menunjukkan kepada dosen bahwa Anda tidak hanya memakai komponen dasar (Tombol, Label, Textfield).

Dashboard Statistik dengan Grafik (JFreeChart)

Di menu utama TU, jangan biarkan kosong. Gunakan library JFreeChart (bekerja 100% offline) untuk menggambar Grafik Batang (Bar Chart) atau Grafik Lingkaran (Pie Chart).
Contoh: Grafik jumlah peminjaman minggu ini, atau porsi perbandingan barang yang tersedia vs dipinjam. Dosen visual sangat menyukai representasi data dalam bentuk grafik.

QR Code Offline di dalam Laporan (ZXing + JasperReports)

Anda sudah pakai JasperReports untuk cetak struk PDF. Tambahkan elemen Barcode/QR Code di sudut struk yang berisi "ID Transaksi".
Ini tidak butuh internet sama sekali, library ZXing akan generate gambar QR secara lokal. Saat demo, Anda bisa pura-pura men-scan QR tersebut pakai HP atau barcode scanner USB.

🔥 Level 3: Interaksi Real-Time & Event Handling (Nilai Plus)

Meskipun offline (berjalan di localhost / 1 jaringan WiFi yang sama), Anda tetap bisa membuat interaksi yang dinamis.

Auto-Refresh Tabel Tanpa Klik (javax.swing.Timer)

Biasanya di Java, kalau ada mahasiswa kirim pengajuan, TU harus klik tombol "Refresh Data" agar tabelnya terupdate.
Cara "Wah": Pasang Timer di background form TU yang mengeksekusi fungsi tampilData() setiap 5 detik secara diam-diam. Saat demo, letakkan window Mhs dan window TU berdampingan. Saat Mhs klik "Kirim", tabel di window TU langsung terupdate otomatis sepersekian detik kemudian tanpa diklik!

Validasi Input Real-time (Event KeyReleased)

Tunjukkan bahwa aplikasi Anda "cerdas" secara visual. Saat mahasiswa mengetik "Jam Mulai" dan "Jam Selesai", beri event KeyReleased. Jika jam selesai lebih kecil dari jam mulai, textfield otomatis berubah warna jadi merah dan muncul tulisan peringatan secara langsung tanpa harus menunggu tombol "Simpan" ditekan.

💾 Bonus: Peningkatan Sisi Database (MySQL) & Logika Backend

Untuk mendapatkan apresiasi ekstra dari dosen (terutama jika ada penguji yang paham database), tambahkan teknik-teknik manipulasi database tingkat lanjut ini:

Otomatisasi Stok dengan TRIGGER MySQL

Masalah: Mengurangi/menambah stok barang saat peminjaman menggunakan kode Java berisiko rentan error (misal aplikasi ditutup tiba-tiba).
Solusi "Wah": Buat Trigger di MySQL. Ketika Admin mengubah status pengajuan menjadi ACC, biarkan database yang otomatis memotong jumlah stok_total di tabel barang. Begitu juga saat statusnya "Dikembalikan", stok otomatis bertambah. Ini menunjukkan pemahaman Database Management System (DBMS) yang sangat solid.

Penyederhanaan Query dengan VIEW

Masalah: Untuk membuat laporan (Report Jasper), Anda biasanya harus membuat query SELECT dengan banyak JOIN (antara tb_transaksi, tb_mahasiswa, tb_barang). Kodenya menjadi panjang dan sulit di-debug di Java.
Solusi "Wah": Buat sebuah Virtual Table atau VIEW di MySQL (contoh: v_rekap_peminjaman). Di dalam Java dan JasperReports, Anda cukup memanggil SELECT * FROM v_rekap_peminjaman.

Konsep Soft-Delete (Anti Data Hilang)

Masalah: Jika Admin menghapus data Mahasiswa atau Barang menggunakan DELETE FROM, maka data historis transaksi mahasiswa/barang tersebut akan ikut hilang atau mengalami error karena aturan Foreign Key.
Solusi "Wah": Jangan gunakan fungsi Delete permanen. Tambahkan kolom is_active (tipe BOOLEAN / INT) di tabel Master. Saat Admin klik "Hapus", sebenarnya Java hanya melakukan UPDATE tb_barang SET is_active = 0. Di tabel GUI, tampilkan hanya data dengan is_active = 1. Cara ini mempertahankan riwayat pelaporan.

Penerapan Transaction Control (Commit & Rollback) di Java

Saat melakukan proses rumit (contoh: Insert data ke tb_transaksi sekaligus Update status di tb_ruangan), gunakan konsep ACID (Atomicity). Di koneksi Java Anda, gunakan conn.setAutoCommit(false). Jika semua eksekusi query berhasil, jalankan conn.commit(). Jika ada satu saja yang gagal/error di tengah jalan, tangkap di catch dan jalankan conn.rollback(). Dosen akan terkesima karena ini adalah praktik standar industri untuk mencegah kebocoran data.

🛠️ Rekomendasi Prioritas untuk Demo

Jika waktu mepet, cukup implementasikan 3 hal utama ini untuk memastikan presentasi Anda memukau baik dari segi depan (Visual) maupun belakang (Database):

  • FlatLaf (Visual): Ubah tema UI menjadi modern / dark mode. Sangat terasa perubahannya.
  • Timer Auto-Refresh (Interaktif): Demo-kan window Mhs yang submit data, dan window TU yang tiba-tiba ter-update datanya tanpa disentuh.
  • Trigger Stok (Database): Tunjukkan bahwa saat di-ACC, stok barang otomatis berkurang oleh Trigger MySQL, bukan oleh query UPDATE Java biasa. Ini akan jadi "kartu as" saat sesi tanya jawab logika database.
