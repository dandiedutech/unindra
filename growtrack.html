Panduan Pengoperasian GrowTrack

Panduan teknis bagi Admin, Bidan, dan Petugas Posyandu untuk optimalisasi pencatatan rekam medis digital balita berdasarkan regulasi Permenkes No.2/2020.

1. Pengenalan & Hak Akses

Sistem GrowTrack dirancang untuk membagi tanggung jawab kerja di lingkungan Posyandu secara jelas. Otorisasi peran (Role) menentukan fitur apa saja yang dapat dilihat dan dimodifikasi untuk menjamin akurasi dan kerahasiaan data balita.

Admin

Memiliki akses menyeluruh, termasuk manajemen database dasar dan pembuatan akun bagi personil lainnya.

Otorisasi Penuh

Petugas / Bidan

Melakukan pendaftaran balita, imunisasi, dan pencatatan pemeriksaan. Menu konfigurasi sistem disembunyikan otomatis.

Operasional Medis

Kasir

Mendata operasional keluar masuknya biaya Pemberian Makanan Tambahan (PMT) serta administrasi non-medis.

2. Alur Kerja (Workflow)

Agar pengoperasian database tetap terjaga integritasnya (mencegah data yatim/invalid), pengisian formulir harus dijalankan secara berurutan:

1

Registrasi Orang Tua

Input data identitas Ayah, Ibu, Pekerjaan, dan Alamat di Form Orang Tua.

2

Pendaftaran Balita

Mendaftarkan anak dan menghubungkannya dengan ID Orang Tua yang sudah dibuat.

3

Pelayanan Medis

Pengukuran BB, TB, LK, serta pemberian Imunisasi / Vaksin berkala di posyandu.

4

Evaluasi & Laporan

Menghasilkan laporan PDF untuk diserahkan kepada pihak Kelurahan/Puskesmas.

4. Manajemen Master Data

A. Form Orang Tua (Ortu)

Formulir pendaftaran induk wali dari bayi/balita. Kolom input terdiri dari:

Kolom Input Tipe Data Keterangan & Batasan
ID Orang Tua Varchar (10) Dihasilkan otomatis oleh sistem dengan pola ORT-xxx (Read-Only).
Nama Ibu Varchar (100) WAJIB DIISI. Identitas penanggung jawab tumbuh kembang anak.
Nama Ayah Varchar (100) Bersifat opsional/tambahan.
No. HP Varchar (15) Wajib berupa angka. Sistem akan mendeteksi kesalahan jika diketik huruf.
Alamat Text Domisili tempat tinggal sekarang untuk keperluan monitoring dinas.

B. Form Balita

Setiap balita harus dihubungkan secara relasional dengan data orang tuanya yang telah terdaftar.

Kolom Input Tipe Data Keterangan & Aturan
Pilih Orang Tua Dropdown Autocomplete Ketik nama orang tua, lalu pilih daftar pencarian yang muncul (misal: ORT-001 - Siti Aminah).
Jenis Kelamin Enum (L, P) MUTLAK. Menentukan kurva acuan normalitas gizi (Laki-laki atau Perempuan).
Tanggal Lahir Date Dipilih via kalender. Digunakan oleh sistem untuk melacak konversi usia bulan pada saat penimbangan.

5. Pemeriksaan Tumbuh Kembang (Modul Inti)

Petunjuk Pengoperasian Input Real-Time

Saat bidan melakukan input pada kolom Berat Badan (BB) dan Tinggi Badan (TB), harap tekan tombol Enter atau Tab untuk memicu rekalkulasi otomatis status gizi dan catatan bidan secara seketika.

Rincian Kolom Formulir Pemeriksaan

Nama Input Format Input Cara Mengisi & Responsivitas
ID Periksa Varchar (15) Otomatis diisi oleh sistem (pola: PRX-001) untuk meminimalkan salah ketik nomor antrean.
Pilih Pasien Balita Autocomplete Text Ketik minimal 3 huruf nama anak, kotak pilihan akan mengerucut dan tampil di layar.
Berat Badan (BB) Double (Decimal) Ketik dengan desimal menggunakan pemisah titik (cth: 9.8). Menentukan rasio BB/U dan BB/TB.
Tinggi Badan (TB) Double (Decimal) Ketik tinggi anak (cth: 76.5). Mempengaruhi nilai TB/U dan perhitungan stunting.
Lingkar Kepala (LK) Double (Decimal) Ketikkan nilai ukuran lingkar kepala balita. Sistem akan mengalkulasi status lingkar kepala. Terbaru

6. Khusus: Panduan Lingkar Kepala (LK)

Pengukuran Lingkar Kepala (LK) merupakan bagian sangat penting dari pemantauan tumbuh kembang anak usia 0-5 tahun (60 bulan) guna mendeteksi gangguan ukuran otak sejak dini. Standar WHO mengklasifikasikan lingkar kepala anak menjadi 3 kategori:

1. Mikrosefali (Terlalu Kecil)

Nilai LK berada dibawah -2 Standar Deviasi (-2 SD) kurva pertumbuhan. Indikasi perkembangan otak terhambat.

2. Normal

Nilai LK di antara -2 SD sampai +2 SD. Struktur kepala tumbuh dengan proporsi yang sempurna.

3. Makrosefali (Terlalu Besar)

Nilai LK berada diatas +2 Standar Deviasi (+2 SD). Risiko hidrosefalus atau penumpukan cairan.

7. Jadwal & Vaksinasi

Membantu melacak jadwal imunisasi mendatang agar bayi tidak melewatkan vaksin wajib nasional.

  • Input ID Balita: Pilih balita yang akan dijadwalkan menerima suntikan.
  • Tanggal Imunisasi Berikutnya: Diatur agar sistem secara cerdas menampilkan pengingat di layar beranda.
  • Status Bayar/Layanan: Membagi pendaftaran berdasarkan rujukan BPJS atau berbayar umum.

8. Konsumsi PMT Operasional

Melacak penggunaan anggaran belanja dapur dalam penyediaan Pemberian Makanan Tambahan (PMT) Posyandu.

  • Nama Menu: Tulis makanan bergizi yang dibagikan (misal: Sup Telur Puyuh).
  • Biaya Pengeluaran: Masukkan angka nominal bersih.
  • Penerima Manfaat: Total porsi piring yang terdistribusi kepada balita hadir.

9. Laporan PDF & Pengaturan

Ekspor Laporan PDF

Ekspor PDF diatur menggunakan pustaka JasperReports untuk menghasilkan lembar kop surat dinas secara resmi.

  • Filter Rentang Tanggal: Cetak hanya riwayat penimbangan di tanggal tertentu.
  • Pencegahan Duplikasi: ID balita unik akan digunakan sebagai index barcode laporan.

Konfigurasi Pengaturan

Khusus untuk admin, menu pengaturan dapat mengubah nama posyandu dan logo direktori lokal untuk diaplikasikan ke kop surat laporan secara real-time.

UPDATE pengaturan SET nama_posyandu = ?, logo_path = ? WHERE id = 1;
